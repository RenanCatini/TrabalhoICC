-Como utilizar o Git-

Esse site irá ensinar como usar o sistema de controle
do Git e o repositório Github.

Introdução

Git é uma ferramenta que ajuda a organizar e controlar as mudanças no código de um projeto.
Ela permite que várias pessoas trabalhem juntas sem bagunçar nada, e ainda guarda um histórico
de todas as mudanças, facilitando voltar atrás se algo der errado.

Git

Segue a descrição e explicação dos seguintes comandos do git:

  1. init:
    Este comando é usado para criar um novo repositório Git em um diretório específico.
    Ele inicializa um diretório como um repositório Git, permitindo que você comece a acompanhar
    as alterações de arquivos e faça commits.

  2. clone:
    O comando clone é usado para criar uma cópia de um repositório Git existente em um novo diretório.
    Ele copia todo o histórico de commits, ramos e arquivos do repositório original para o novo diretório.

  3. branch:
    Este comando é usado para criar, listar ou excluir ramos no repositório Git.
    Um ramo no Git é simplesmente uma referência móvel para um commit.

  4. checkout:
    O comando checkout é usado para alternar entre ramos ou restaurar arquivos de um commit específico.
    Ele pode ser usado para criar um novo ramo ou alternar entre ramos existentes.

  5. status:
    Este comando mostra o estado atual do repositório Git. Ele exibe informações sobre arquivos não rastreados,
    arquivos modificados e o estado dos ramos.

  6. diff:
    O comando diff é usado para exibir as diferenças entre dois pontos de dados Git. Isso pode ser entre commits,
    entre o diretório de trabalho e o índice, ou entre o índice e o commit atual.

  7. add:
    Este comando é usado para adicionar alterações feitas em arquivos ao índice do Git.
    Isso prepara as alterações para serem incluídas no próximo commit.

  8. commit:
    O comando commit é usado para salvar as alterações feitas no repositório Git.
    Ele cria um novo commit contendo as alterações preparadas com o comando add.

  9. push:
    Este comando é usado para enviar commits locais para um repositório remoto.
    Ele transfere as alterações locais para o repositório remoto especificado.

  10. pull:
    O comando pull é usado para recuperar as alterações de um repositório remoto e mesclá-las com o repositório local.
    Ele é uma combinação dos comandos fetch e merge.

  11. revert:
    Este comando é usado para desfazer um commit específico,
    criando um novo commit que desfaz as alterações introduzidas pelo commit original.

  12. merge:
    O comando merge é usado para mesclar alterações de um ramo para outro.
    Ele combina o histórico de dois ramos diferentes.

  13. stash:
    Este comando é usado para temporariamente armazenar alterações locais que ainda não estão prontas para serem commitadas.
    Isso permite que você trabalhe em outra tarefa sem ter que fazer commit das alterações atuais.

  14. rm:
    O comando rm é usado para remover arquivos do índice e do diretório de trabalho.
    Isso também significa que o arquivo não será mais rastreado pelo Git.

  15. config:
    Este comando é usado para configurar variáveis de configuração do Git.
    Isso pode incluir informações de usuário, preferências de formatação, configurações de repositório, etc.

  16. reset:
    O comando reset é usado para desfazer alterações no índice e/ou no diretório de trabalho.
    Ele pode ser usado para desfazer commits, mover a cabeça para um commit específico, ou desfazer alterações no índice.

  17. remote:
    Este comando é usado para gerenciar repositórios remotos. Ele permite que você adicione,
    remova e liste repositórios remotos associados ao seu repositório local.

  18. fetch:
    O comando fetch é usado para baixar commits, arquivos e referências de um repositório remoto para o repositório local.
    Isso não mescla automaticamente as alterações com o repositório local.

  19. show:
    Este comando é usado para exibir informações sobre vários objetos Git,
    como commits, tags, árvores, etc.

  20. help:
    O comando help é usado para obter informações de ajuda sobre os comandos Git.
    Ele exibe informações sobre como usar os comandos Git e suas opções.

  21. rebase:
    O comando rebase é usado para alterar a base de um ramo. Isso move os commits de um ramo para o início de outro ramo,
    reescrevendo o histórico de commits.

  22. blame:
    Este comando é usado para exibir quem modificou cada linha de um arquivo e em qual commit a modificação foi feita.

  23. tag:
    O comando tag é usado para criar, listar ou excluir tags no repositório Git.
    As tags são usadas para marcar commits específicos como versões ou pontos de referência importantes.

Github

Para configurar o GitHub, siga estes passos:

1. Crie uma conta no GitHub.
2. Crie um novo repositório.
3. Clone o repositório em seu computador local usando git clone.
4. Adicione e modifique arquivos no repositório local.
5. Adicione arquivos ao índice do Git usando git add.
6. Confirme as alterações com git commit.
7. Envie as alterações para o repositório remoto no GitHub com git push.
